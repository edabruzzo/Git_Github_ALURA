<p><i><b>Flyweight</b></i> é um <a href="/wiki/Padr%C3%B5es_de_projeto_de_software" class="mw-redirect" title="Padrões de projeto de software">padrão de projeto de software</a> apropriado quando vários objetos devem ser manipulados em memória sendo que muitos deles possuem informações repetidas. Dado que o recurso de <a href="/wiki/Mem%C3%B3ria_(computador)" class="mw-redirect" title="Memória (computador)">memória</a> é limitado, é possível segregar a informação repetida em um objeto adicional que atenda as características de imutabilidade e comparabilidade (que consiga ser comparado com outro objeto para determinar se ambos carregam a mesma informação).
</p><p>Um exemplo é o <a href="/wiki/Processador_de_texto" title="Processador de texto">processador de texto</a>. Cada caractere representa um objeto que possui uma família de fonte, um tamanho de fonte e outras informações sobre o símbolo. Como imaginado, um documento grande com tal <a href="/wiki/Estrutura_de_dados" title="Estrutura de dados">estrutura de dados</a> facilmente ocuparia toda a memória disponível no sistema. Para resolver o problema, como muitas dessas informações são repetidas, o <i>flyweight</i> é usado para reduzir os dados. Cada objeto de caractere contém uma referência para outro objeto com suas respectivas propriedades.
</p>
<h2><span class="mw-headline" id="Imutabilidade_e_igualdade">Imutabilidade e igualdade</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flyweight&amp;veaction=edit&amp;section=1" class="mw-editsection-visualeditor" title="Editar secção: Imutabilidade e igualdade">editar</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Flyweight&amp;action=edit&amp;section=1" title="Editar secção: Imutabilidade e igualdade">editar código-fonte</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Para que seja seguro o compartilhamento entre clientes e <a href="/wiki/Thread_(ci%C3%AAncia_da_computa%C3%A7%C3%A3o)" class="mw-redirect" title="Thread (ciência da computação)">threads</a>, objetos Flyweight devem ser imutáveis. Objetos de Flyweight são por definição objetos de valor. A identidade de instância do objeto não possui nenhuma consequência se dois valores são considerados iguais.
</p>
